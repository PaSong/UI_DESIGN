UI 디자인 가이드

01.모바일에서는 모달 상태의 경고창 alert상단 우측 "X"버튼은 노출하지 않습니다.

02.풀프레임 대화창Dialog에 "x"버튼 (닫기 버튼)은 왼쪽 상단을 권장

03.긍정적인 액션 버튼은 우측에 배치함

04.입력을 완료하지 않은 상태에서 완료 버튼(보내기/로그인)은 오류를 방지하기 위해서 비 활성화를 권장함

05.입력 내용에 맞는 키보드를 제공해야 함

06.검색 필드 우측에 검색 버튼은 추천하지 않음

07.스크롤을 고려하여 콘텐츠의 그리드를 화면 하단에 정확하게 정렬하지 마십시오.

08.텝은 중첩되지 않아야 합니다.

09.버튼 스타일은 중요도에 맞게 사용해야 합니다.

10.시각적 노이즈는 최소화해야 합니다.

11.내비게이션 바(앱바)에서 아이콘과 텍스트 레이블을 함께 사용하지 마십시오.

12.키보드를 통해 많은 정보를 입력해야 하는 대화창에서 [완료]를 의미하는 버튼은 상단 내비게이션 바 우측을 권장합니다.

13.회원가입 또는 로그인 시 입려한 비밀번호를 가리거나 볼 수 있는 옵션을 제공하세요.

14.내비게이션 드로워는 1단으로 하길 권장합니다.

15.사용자가 입력하는 내용이 형식에 맞을 경우 실시간 피드백을 해주세요.

16.버튼에 사용한 스타일을 다른 요소(섹션 제목)에 사용하지 마세요.

17.수량 선택 기능은 오류 방지를 위해 드롭다운보다 스텝퍼 사용을 권장합니다.

18.텍스트 필드에 입력 한 텍스트를 모두 지울 수 있는 Clear button은 필수 적으로 제공해야 합니다.

19.라디오 버튼은 언제나 하나의 항목이 기본으로 선택되어있아애 힙니다.

20.옵션 선택을 완료하지 않은 상태에서 오류를 방지를 위해 완료 버튼은 비 활성화를 권장합니다.

21.'예','아니오'같이 두 개의 옵션만 있는 경우 하나의 체크박스 또는 토글스위치를 사용합니다.

22.키보드에 중요한 액션 버튼이 가리지 않아야 합니다.

23.모바일에 팝업을 권장하지 않지만 꼭 필요하다면 풀프레임 팝업을 고려해보세요.

24.모바일에서 좌우 슬라이드 되는 콘텐츠는 가로 그리드에 딱 맞출 필요는 없습니다.

25.메인 페이지의 슬라이더 이미지는 3~5개가 적정합니다.

26.라디오 버튼은 소팅 기능에 사용할 수 없습니다.

27.라디오 버튼 옵션 내에 또 다른 옵션을 넣지 않습니다.

라디오 버튼 항목 안에 다른 라디오 버튼이나 체크박스나 입력 필드 등 다른 옵션을 만들지 않아야 함.

모든 라디오 버튼 옵션은 선택과 관련된 하나의 레벨로 디자인되어야 합니다.

28.링크 텍스트에 밑줄은 사용하지 않습니다.

29.선택 옵션이 5개 이하일 경우 드롭다운보다는 옵션을 노출하게는 좋습니다.

일반적으로 7개 이상의 옵션일 경우 드롭다운 사용을 권장합니다.

30.탭 바의 항목은 최대 5개로 제한됩니다.

31.경고창의 버튼은 테두리가 없는 텍스트 버튼 사용을 권장합니다.

32.테두리가 있거나 면으로 된 버튼보다는 테두리가 없는 텍스트 버튼을 기본으로 사용하세요.

33.중요한 콜투 액션 버튼의 경우 사용자가 스크롤하더라도 지속적으로 볼 수 있어야 합니다.

34.내비게이션 요소와 UI요소가 콘텐츠보다 더 부각되지 않아야 합니다.

35.삭제와 같이 되돌릴 수 없는 액션의 경우, 다시 한번 확인요청을 합니다.

36.사용자의 노력으로 만들어진 정보는 쉽게 삭제할 수 없게 디자인되어야 합니다.

37.리스트에서 선택 또는 실행 버튼은 불필요합니다.

38.주 액션과 되돌릴 수 없는 액션은 같은 위치에 배치하지 않을 것을 권장합니다.

39.본문의 행간은 150%이상 이어야 읽기가 쉽습니다.

40.내비게이션 바에 너무 많은 컨트롤은 피하세요.

41.한 화면에 뒤로 가기 버튼은 하나여야 합니다.

42.상단 내비게이션 바에 내비게이션 요소의 중복을 피할 것을 권장합니다.

43.뒤로 가기 버튼 옆의 레이블은 이전 화면명입니다.

44.모바일에서는 브레드 스크럼 Breadcrumbs을 사용하지 않습니다.

45.모바일에서 탑 버튼을 사용하지 않아도 됩니다.

46.가독성을 위해서 중앙 정렬은 피하세요.

47.비활성화 요소에 활성화 요소와 동일한 색상을 사용하지 마세요.

48.버튼을 텝했을 때 일어나는 동작과 사용자의 의도와 관련이 없는 [예],[아니오]같은 레이블은 사용하지 않아야 합니다.
